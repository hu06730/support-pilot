# 文件系统

## 1. 文件系统概述

文件系统是操作系统中负责管理和存储文件的子系统。文件是具有符号名的相关信息的集合。

文件的逻辑结构：
- 流式文件：无结构的字节序列
- 记录式文件：由一组记录组成

文件的物理结构：
- 连续分配：文件占用连续的磁盘块
- 链接分配：文件的磁盘块通过指针链接
- 索引分配：使用索引表记录文件的磁盘块

## 2. 文件操作的系统调用

**open(path, flags, mode)**：打开文件，返回文件描述符。

**read(fd, buf, count)**：从文件描述符 fd 读取 count 字节到 buf。

**write(fd, buf, count)**：将 buf 中的 count 字节写入文件描述符 fd。

**close(fd)**：关闭文件描述符。

**lseek(fd, offset, whence)**：设置文件偏移量，用于随机访问。

文件描述符是一个非负整数，用于标识打开的文件。标准输入（stdin）的文件描述符为 0，标准输出（stdout）为 1，标准错误（stderr）为 2。

## 3. 目录操作

**opendir(path)**：打开目录，返回目录流。

**readdir(dir)**：读取目录中的下一个条目。

**closedir(dir)**：关闭目录。

实现 ls 命令：使用 opendir 和 readdir 遍历目录，打印每个文件名。

实现 mkdir 命令：使用 mkdir() 系统调用创建目录。

## 4. 文件复制

文件复制程序的基本逻辑：
1. 以读模式打开源文件
2. 以写模式打开/创建目标文件
3. 循环读取源文件内容并写入目标文件
4. 关闭两个文件

注意事项：如果目标文件已存在，默认会在开头插入内容。需要使用 O_TRUNC 标志截断目标文件。

## 5. 文件目录结构

**单级目录**：所有文件在同一目录下，命名冲突。

**两级目录**：每个用户有独立的目录。

**树形目录**：多级目录结构，支持层次化组织。

**无环图目录**：支持文件共享（硬链接、符号链接）。

**硬链接**：多个目录项指向同一个 inode。删除一个硬链接不影响其他链接。

**符号链接（软链接）**：一个特殊文件，包含指向另一个文件的路径。删除原文件后符号链接失效。

## 6. 文件保护

访问控制矩阵：行表示主体（用户），列表示客体（文件），矩阵元素表示访问权限。

访问控制列表（ACL）：每个文件关联一个 ACL，记录哪些用户有哪些权限。

文件权限：读（r）、写（w）、执行（x），分为所有者、组用户、其他用户三类。

## 7. 文件系统实现

**超级块（Superblock）**：存储文件系统的元数据，如块大小、inode 数量、空闲块数量等。

**inode**：存储文件的元数据（大小、权限、时间戳、数据块指针），不包含文件名。

**数据块**：存储文件的实际内容。

**位图法**：用位图记录空闲块的分配状态。

**文件分配表（FAT）**：用表记录每个块的下一个块号。

## 8. 虚拟文件系统（VFS）

VFS 是一个抽象层，为不同的文件系统提供统一的接口。用户程序通过 VFS 访问文件，不需要关心底层文件系统的具体实现。

VFS 支持的文件系统类型：ext4、NTFS、FAT32、NFS 等。

## 9. IO 模型

**阻塞 IO**：进程发起 IO 请求后阻塞，直到 IO 完成。

**非阻塞 IO**：进程发起 IO 请求后立即返回，需要轮询检查 IO 状态。

**IO 多路复用**：使用 select/poll/epoll 同时监控多个文件描述符。

**异步 IO**：进程发起 IO 请求后立即返回，IO 完成后通知进程。

select 的缺点：文件描述符数量有限（通常 1024），每次调用需要遍历所有描述符。

epoll 的优点：没有文件描述符数量限制，事件驱动，效率高。
