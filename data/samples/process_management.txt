# 进程管理

## 1. 进程基础概念

进程是程序的一次执行实例，是系统进行资源分配和调度的基本单位。每个进程拥有独立的地址空间、代码段、数据段和堆栈段。

进程与程序的区别：程序是静态的代码集合，进程是动态的执行过程。一个程序可以对应多个进程。

## 2. fork 系统调用

fork() 是 UNIX/Linux 系统中用于创建新进程的系统调用。调用 fork() 后，操作系统会创建一个与父进程几乎完全相同的子进程。

fork() 的返回值：
- 在父进程中返回子进程的 PID（正数）
- 在子进程中返回 0
- 创建失败时返回 -1

fork() 创建的子进程继承了父进程的代码段、数据段、堆栈、文件描述符等。父子进程共享代码段但拥有独立的数据段。

连续调用 fork() 会创建指数级增长的进程数。例如连续三次 fork() 会创建 2^3 = 8 个进程。

## 3. 进程状态

进程的三种基本状态：
- **就绪态（Ready）**：进程已获得除 CPU 之外的所有必要资源，等待 CPU 调度
- **运行态（Running）**：进程正在 CPU 上执行
- **阻塞态（Blocked）**：进程因等待某个事件（如 I/O 完成）而暂停执行

状态转换：
- 就绪 → 运行：被调度程序选中，获得 CPU
- 运行 → 就绪：时间片用完或被高优先级进程抢占
- 运行 → 阻塞：请求 I/O 或等待事件
- 阻塞 → 就绪：等待的事件发生

## 4. 孤儿进程和僵尸进程

**孤儿进程**：父进程先于子进程结束，子进程成为孤儿进程。系统会将孤儿进程的父进程设为 init 进程（PID 为 1）。

**僵尸进程**：子进程执行完毕但父进程没有调用 wait() 回收其退出状态，子进程成为僵尸进程。僵尸进程不占用 CPU 但占用进程表条目。

避免僵尸进程的方法：
- 父进程调用 wait() 或 waitpid() 回收子进程
- 使用 signal(SIGCHLD, SIG_IGN) 忽略子进程退出信号
- 使用 double-fork 技术

## 5. 进程调度算法

**FCFS（先来先服务）**：按进程到达顺序执行，简单但可能导致长作业阻塞短作业。

**SJF（短作业优先）**：选择估计运行时间最短的进程执行，平均等待时间最优但可能导致长作业饥饿。

**时间片轮转调度**：每个进程分配一个时间片，时间片用完后切换到下一个进程。时间片过大退化为 FCFS，过小则上下文切换开销大。

**优先级调度**：按进程优先级调度，可能导致低优先级进程饥饿。

**多级反馈队列调度**：结合时间片和优先级，设置多个就绪队列，新进程进入最高优先级队列，时间片用完则降低优先级。

## 6. 进程同步

**临界区**：访问共享资源的代码段，同一时刻只能有一个进程在临界区内执行。

**互斥**：保证同一时刻只有一个进程访问共享资源。

**信号量**：由 Dijkstra 提出的同步机制，包括 P 操作（wait）和 V 操作（signal）。

**死锁**：两个或多个进程互相等待对方持有的资源，导致所有进程都无法继续执行。

死锁产生的四个必要条件：
1. 互斥条件：资源只能被一个进程独占
2. 占有并等待：进程持有资源的同时等待其他资源
3. 不可剥夺：已分配的资源不能被强制剥夺
4. 循环等待：存在进程的循环等待链

死锁的预防方法：破坏四个必要条件之一。

死锁的避免方法：银行家算法，通过安全序列判断分配是否安全。

## 7. 线程

线程是进程中的执行单元，是 CPU 调度的基本单位。同一进程的多个线程共享进程的地址空间和资源。

线程与进程的区别：
- 进程是资源分配的基本单位，线程是 CPU 调度的基本单位
- 进程切换开销大，线程切换开销小
- 进程间通信需要 IPC 机制，线程间可直接共享内存

用户级线程由用户空间的线程库管理，内核不可见。内核级线程由内核管理，内核可见。
